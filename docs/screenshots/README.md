# Screenshots

Drop three PNGs (or JPGs — update the extensions in the root README if you do) here
so the **Screenshots** table in the main [`README.md`](../../README.md) renders:

| Filename | What to capture |
|---|---|
| `upload.png` | The landing / upload screen (hero, drop zone, sample chips). |
| `results.png` | A results dashboard — run **Analyze sample BOM** for the polished view (score banner, scaled impact, design fixes, AI summary). |
| `review-panels.png` | The **Confirm the materials we matched** + **Add the missing masses** panels — load the **Messy** sample to trigger them. |

Tips for clean shots:
- Use a wide browser window (~1400px) so the cards aren't cramped.
- Light theme (the default warm-paper canvas) reads best.
- Crop to the app content; trim browser chrome.
